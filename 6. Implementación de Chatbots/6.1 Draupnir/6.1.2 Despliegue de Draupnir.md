
En este documento se detalla el proceso técnico para la instalación y despliegue del bot de moderación Draupnir.

Draupnir hay que desplegarlo como un contenedor Docker que se comunica con la API de Synapse, actuando como un usuario con permisos de administrador.

1. **Creamos un usuario y sala personalizada para el bot:**

 **Para el usuario**

> Para crear el usuario, como el inicio/registro manual está desactivado, la única manera de crear un usuario que no pase el filtro SAML de la UGR es mediante la pestaña de **Synapse-Admin**.  
> Si no se sabe acceder a esta ventana, se recomienda revisar el Caso de uso `5.2.1 Acceso a Synapse-Admin`, donde se explica.


1. Una vez dentro, en el listado de usuarios, añadimos uno nuevo.
2. Es muy importante que, para el correcto funcionamiento del bot, en el momento de su creación se le deleguen **permisos de administración** (_Server Administrator_).
3. Cuando esté creado, obtendremos su **Access Token** y lo guardaremos para después.

**Para la sala**

Para comunicarnos con Draupnir necesitamos de una sala directa con él.

> **NOTA IMPORTANTE:** La sala de comandos **NO debe estar cifrada (E2EE)**. Draupnir no posee accedo a las claves criptográficas de los usuarios o de las salas privadas. Es por esto que si nos comunicamos con el a través de una sala con cifrado no nos entenderá y nos enviará un mensaje de error.

1. Creamos una Sala privada (En nuestro caso `draupnir-sala` dentro de el espacio `salas-bots`)
2. Desactivamos el Cifrado de Extremo a extremo.
3. Vamos a los ajustes avanzados de la sala para obtenemos el **ID interno de la sala** y lo guardamos para después.

----

2. **Creamos y añadimos los archivos de configuración**

Dentro de el servidor tenemos un directorio dedicado a los bots. Así que dentro de este creamos un espacio y directorio para la configuración de draupnir:

``` bash
mkdir -p bots/draupnir/config
mkdir -p bots/draupnir/data
nano bots/draupnir/config/production.yaml
```

Dentro del archivo de producción añadimos el token del usuario y el id de la sala que hemos guardado anteriormente, además de la siguiente información:

``` YAML
# Token de acceso del usuario
accessToken: "syt_TU_ACCESS_TOKEN_AQUI"

#URL interna de Synapse 
homeserverUrl: "http://synapse:8008"

#ID interno de la sala
managementRoom: "!tu_id_de_sala:matrix.ugr.es"

#Ruta de almacenamiento de datos dentro del contenedor
dataPath: "/data"
```

3. **Integración con el Docker Composer**

Añadimos el siguiente bloque a nuestro archivo docker-compose.yml, para que descargue y configure los directorios que hemos creado antes para su correcto funcionamiento:

``` YAML
draupnir:
    image: gnuxie/draupnir:latest
    container_name: draupnir_bot
    restart: unless-stopped
    volumes:
	  - ./bots/draupnir/config:/data/config:ro
      - ./bots/draupnir/data:/data:rw
    command: ["bot", "--draupnir-config", "/data/config/production.yaml"]
    depends_on:
      - synapse
```

4. **Lo ponemos en marcha:**

Una vez guardados los archivos, ya podemos levantar el contenedor:

```bash
sudo docker compose up -d draupnir
```

Dicho esto esto ya solo nos quedaría invitar al usuario bot  (`@bot.draupnir:matrix.ugr.es`) a nuestra sala creada en el primer paso.

Una vez invitado si todo estuviera bien configurado por sí solo el aceptará la invitación y accederá al chat.

Después de esto, el mismo nos pedirá que le demos permisos de administrador en la misma sala, una vez se los demos, recargamos el bot y ya podremos comenzar a utilzarlo.

``` bash
sudo docker compose restart draupnir
```
