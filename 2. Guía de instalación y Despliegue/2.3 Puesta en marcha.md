**1. Generación de la configuración inicial:** Synapse requiere generar un entorno base antes de arrancar, además de establecer el directorio donde vamos a almacenar varios archivos calves de este: 

``` bash
sudo docker run -it --rm \
    -v "$PWD/data:/data" \
    -e SYNAPSE_SERVER_NAME=matrix.ugr.es \
    -e SYNAPSE_REPORT_STATS=no \
    matrixdotorg/synapse:latest generate
```

**2. Ajuste de permisos (UID 991):** Este paso es muy importante, y es que la imagen oficial de Synapse corre bajo un usuario interno dentro de la máquina con ID 991. Si no cambiamos el propietario de la carpeta data, el contenedor no tendrá permisos para escribir en la base de datos y tendremos fallos al arrancar:


```bash
sudo chown -R 991:991 ./data
```

**3. Creación manual de los archivos de orquestación:** Tras generar la base, teenemos que crear y configurar manualmente los archivos que definen el servicio. En este punto se editaron archivos que describiremos más tarde:

- Creación del `docker-compose.yml` para gestionar los contenedores de Synapse y Element.
- Edición del `data/homeserver.yaml` para ajustar listeners y bases de datos.
- Creación del `element-config.json` para la personalización del front de la UGR.

**4. Despliegue de los contenedores:** 

``` bash
sudo docker compose up -d
sudo docker ps
```

**5. Configuración del Proxy Inverso (Nginx):**  Para que el tráfico exterior llegue a los contenedores que hemos desplegado tenemos que instalar y configurar Ngynx:

```bash
sudo dnf install -y nginx
sudo systemctl start nginx
sudo systemctl enable nginx
```

Es un importante crear el archivo de configuración de "/etc/nginx/conf.d/matrix.ugr.es.conf" y realizar un enlace simbólico a este para activar el sitio. 


```bash
sudo ln -s /etc/nginx/sites-available/matrix.ugr.es.conf /etc/nginx/sites-enabled/
sudo nginx -t
sudo systemctl reload nginx
```


**Nota sobre Certificados:** Se utilizaron los certificados oficiales de la Universidad de Granada almacenados localmente, configurando las rutas de `ssl_certificate` y `ssl_certificate_key` directamente en el archivo de Nginx, garantizando la identidad de `matrix.ugr.es`.

**6. Apertura del Firewall:** Finalmente tenemos que habilitar el paso de tráfico web en el firewall de la máquina:


```bash
sudo firewall-cmd --permanent --add-service=http
sudo firewall-cmd --permanent --add-service=https
sudo firewall-cmd --reload
```
