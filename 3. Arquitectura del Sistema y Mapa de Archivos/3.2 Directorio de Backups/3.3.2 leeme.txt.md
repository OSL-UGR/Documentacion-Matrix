Este archivo de texto plano se dejó en la carpeta para que cualquier administrador que no haya llegado a este punto de la documentación y que acceda por SSH al servidor sepa exactamente qué esta viendo y como operar con los backups. 

Este archivo tiene la siguiente estructura:

``` txt
Sistema de copias de seguridad (backups) para el  matrix

- ¿QUÉ ES Y QUE CONTIENE ESTA CARPETA?

  Esta carpeta contiene copias de seguridad semanales comprimidas sel servidor Matrix (Synapse) y su configuración. 


- ¿COMO ESTÁ AUTOMATIZADO?

  Los backups se realizan automáticamente todos los días a las 3AM, mediante una tarea programada con crontab. Estos archivos
  se mantienen en esta carpeta con un máximo de 7 días, evitando llenar el espacio con backups obsoletos.

- ¿COMO DEPURO LA INFORMACIÓN?

  - Para visitar el comando que ejecuta cron todas las noches con persmisos de usuario, podemos ejecutar "sudo crontab -l"
  - La ruta absoluta del archivo que se encarga de ejecutar cron se encuentra en "/home/svc/backup_matrix.sh"
  - Dentro de este mismo directorio se almacena un archivo "log_cron.txt" que va almacenando los logs de las ejecuciones 
    del cron si esque hubiese algún error, para depurar la salida de este es interesante visitarlo.

- ¿Que información contienen los backups?

  - "homeserver.db": La base de datos (SQLite) con loda la información de usuarios, salas, mensajes etc 
  - "homeserver.yaml:" Configuración técnica del servidor.
  - *.key / *.crt: Claves de cifrado y certificados (MUY IMPORTANTES).
  - "element-config.json:" Personalización visual del frontend (Element).
```


Dentro de la información de este archivo hay que resaltar que:

- Los backups se realizan automáticamente todos los días a las 3AM, mediante una tarea programada con **crontab**.
- Para una **depuración** e identificar si **hubiese algún problema** siempre es interesante ejecutar el comando `sudo crontab -l` o bien revisar el archivo de logs `log_cron.txt`.
