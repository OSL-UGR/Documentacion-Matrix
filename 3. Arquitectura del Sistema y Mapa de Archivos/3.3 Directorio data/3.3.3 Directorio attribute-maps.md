
Dentro de este directorio encontramos un archivo muy importante llamado `ugr.py`. Su función principal es actuar como diccionario de traducción entre Proveedor de Identidad (IdP) de la UGR y lo que espera recibir el motor interno de Synapse (PySAML2). Este archivo tiene la siguiente forma:

``` python
MAP = {
    "identifier": "urn:oasis:names:tc:SAML:2.0:attrname-format:uri",
    "fro": {
        'mail': 'uid'
    },
    "to": {
        'uid': 'mail'
    }
}
```

La línea del identifier es importante ponerla, le indica a Synapse que los atributos que van a llegar utilizan el formato de identificador estándar de SAML 2.0.

Los campos de `fro` y `to` también son esenciales. El servidor de la UGR  envía la información del usuario etiquetando su correo electrónico bajo el nombre de la variable `mail`. Sin embargo Synapse trabaja internamente buscando un identificador con el nombre de `uid`. Este bloque de código se encarga de traducir esta comunicación.

Sin este archivo cuando intentemos interceptar el correo del usuario desde `saml_mapping.py` Python nos devolvería un valor nulo generando dentro del script una excepción. Es decir que tendríamos el problema que **ningún usuario de la universidad podría iniciar sesión**.
