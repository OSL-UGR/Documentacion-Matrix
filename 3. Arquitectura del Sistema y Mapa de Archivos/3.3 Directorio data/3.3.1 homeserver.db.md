
Este archivo ubicado en `data/homeserver.db` e sel corazón de la base de datos de nuestro servidor Matrix. Estamos utilizando SQLite como motor que es el que utiliza por defecto Synapse en su configuración. 

Si nos fijamos notaremos que la base de datos está compuesta por 3 archivos distintos. Esto se debe a que SQLite está configurado con **WAL** (Write-Ahead Logging) para aumentar el rendimiento y evitar que la base de datos se bloquee cuando muchos usuarios chateen a la vez:


- **homeserver.db:** Este es el archivo principal. Contiene toda la información de toso los usuarios, las salas creadas, las configuraciones de los perfiles, los mensajes cifrados y el resto de datos que se almacenan en la base de datos. 

- **homeserver.db-wal:** Este archivo es un puente entre la información que recibe la base de datos y el homeserver.db principal. Synapse lo escribe aquí primero de forma super rápida, y ya posteriormente SQLite lo transfiere al archivo principal.

- **homeserver.db-shm:** Es un archivo de memoria compartida que funciona como un índie temporal para que el motor pueda leer el archivo .db-wal a gran velocidad.

**PUNTO IMPORTANTE:** La existencia de estos tres archivos trabajando juntos es la razón por la cual nuestro script de backups `backup_matrixz.sh` ejecuta un `docker compose stop synapse` antes de acceder a los datos para comprimirlos. Si hiciéramos una copia mientras un mensaje essá a medio transefir entre el .db-wal y el .db, la copia de seguridad quedaría corrupta.
