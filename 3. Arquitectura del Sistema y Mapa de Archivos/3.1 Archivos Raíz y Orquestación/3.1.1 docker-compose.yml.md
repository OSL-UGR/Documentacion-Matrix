
El archivo "docker-compose.yml" es el director de la infraestructura. Se encarga de descargar las imágenes, levantar los contenedores de Docker, abrir los puertos necesarios y conectar los volúmenes de datos locales con el interior de los contenedores.

El archivo con la estructura final del proyecto tiene la siguiente forma:

``` YAML
version: '3.8'

services:
  synapse:
    image: matrixdotorg/synapse:latest
    container_name: synapse_server
    restart: unless-stopped
    ports:
      - "0.0.0.0:8008:8008"
    volumes:
      - ./data:/data
    environment:
      - SYNAPSE_CONFIG_DIR=/data
      - SYNAPSE_CONFIG_PATH=/data/homeserver.yaml
      - PYTHONPATH=/data

  element:
    image: vectorim/element-web:latest
    container_name: element_web
    restart: unless-stopped
    ports:
      - "0.0.0.0:8080:80"
    volumes:
      - ./element-config.json:/app/config.json
      - ./favicon.ico:/app/favicon.ico

  synapse-admin:
    image: awesometechnologies/synapse-admin:latest
    container_name: synapse_admin
    ports:
      - "8081:80"
    restart: unless-stopped
    networks:
      - default

networks:
  default:
    ipam:
      driver: default
      config:
        - subnet: 192.168.200.0/24
```

## Explicación de la Configuración

La infraestructura se divide en tres servicios principales y una red personalizada:

### 1. Servicio: `synapse` (Backend)

- **image:** Utilizamos la ultima versión disponible.
- **container_name:** Simplemente especifica el nombre del contenedor.
- **restart: unless-stopped:** Nos garantiza que si la máquina se reiniciase, el servicio se vuelva a arrancar automáticamente.
- **ports:** Establecemos el puerto 8008 que es el puerto por defecto que nos da Synapse hacia el exterior.
- **volumes:** Carga la información de la carpeta "./data" al directorio "/data" dentro de propio contenedor. 
- **environment:**
	- **SYNAPSE_CONFIG_DIR:** hay que decirle el directorio donde se encuentra este archivo, el "homeserver.yml".
	- **SYNAPSE_CONFIG_PATH:** lo mismo que antes pero esta vez el path completo para encontrar el archivo.
	- **PYTHONPATH:** Esta línea es bastante importante, especifica que hay archivos .py adicionales en nuestra carpeta "/data". Gracias a este linea, el contenedor es capaz de leer cualquier scripts que definamos para añadir comportamientos externos. Como es el caso del que hemos añadido nosotros "saml_mapping.py"

### 2. Servicio: `element` (Frontend)

Es el cliente web que utilizan los usuarios para chatear.

- **image**: Utilizamos la última versión disponible.
- **container_name**: Nos dice el nombre del contenedor.
- **restart: unless-stopped**: Lo mismo que para el de synapse, si la maquina se reinicia el servicio arranca automáticamente.
- **ports:** En este caso lo que hacemos es establecer el puerto 8080 de la máquina anfitriona, y redirigimos al puerto 80 que es el puerto interno del contenedor web.
- **volumes:** Ahora en vez de mapear una carpeta entera como en en el synapse, le especificamos a mano nos archivos concretos:
	- **element-config.json:** es el json que nos deja personalizar y sobrescribir configuración por defecto de Element limitando parámetros y añadiendo una estética propia de la universidad. 
	- **favicon.ico:** Para poner un favicon personalizado con el logo de la ugr.


### 3. Servicio: `synapse-admin` (Panel de Gestión)

synapse-admin nos proporciona una interfaz gráfica para la administración de la base de datos del servidor. Aquí podemos gestionar todos los usuarios del sistema, las salas que hay etc

- **ports**: Queda expuesto en el puerto local `8081`.

 Este panel interactúa directamente con la API de Synapse, facilitando enormemente el mantenimiento sin necesidad de acceder a la base de datos a través de la terminal. Hay un caso de uso explicando como entrar y utilizar esta ventana.

### Redes (`networks`)

- Definimos una subred específica en `192.168.200.0/24`para la comunicación interca de los contenedores. Esto es para **evitar conflictos** de enrutamiento que había con los rangos de IPs internas de la universidad.
