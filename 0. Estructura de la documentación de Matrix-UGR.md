## 3. Arquitectura del Sistema y Mapa de Archivos

Este apartado detalla el árbol de directorios del servidor Matrix-UGR ubicado en `/home/svc/`, explicando la función de cada archivo crítico para el despliegue, la personalización y la seguridad.

- **3.1 Archivos Raíz y Orquestación**
    
    - **docker-compose.yml:** Orquestación y vinculación de los contenedores de Synapse y Element.
    - **element-config.json:** Configuración visual, dominios y comportamiento del cliente web.
    - **Configuración Nginx:** Proxy inverso para enrutamiento de tráfico y gestión de certificados SSL.
    - **backup_matrix.sh:** Script Bash automatizado para copias de seguridad seguras en frío.
    
- **3.2 Directorio de Backups**
    
    - **Contenido del directorio:** Almacén de las copias semanales comprimidas en `.tar.gz`.
    - **leeme.txt:** Manual rápido de emergencias in situ para administradores del sistema.
    - **log_cron.txt:** Registro de la salida y posibles errores de la tarea programada `cron`.
    
- **3.3 El Cerebro del Servidor (/data)**
    
    - **homeserver.db:** Base de datos principal (SQLite) y su arquitectura multipropósito (WAL/SHM).
    - **ugr-idp-metadata.xml:** Endpoints y certificados criptográficos del Proveedor de Identidad del CSIRC.
    - **attribute-maps/:** Diccionario para traducir los atributos de la UGR (`mail`) al formato de Matrix (`uid`).
    - **saml_mapping.py:** Script Python personalizado para inyectar prefijos (`e.`) y generar nombres de usuario.
    - **homeserver.yaml:** Configuración maestra del servidor (puertos, federación, SSO y permisos de salas).
    

## **4. Seguridad 

- **El flujo del SSO (SAML):** Viaje de autenticación del usuario desde el CSIRC hasta el acceso a Element.
- **Cifrado de Extremo a Extremo (E2EE):** Gestión de llaves criptográficas y privacidad absoluta frente a administradores.
- **Cifraco con claves en Matrix:** Cómo funciona en más detalle el cifrado que se aplica en los mensajes.


## 5. Casos de Uso 

- **5.1. Operativa y Mantenimiento del Servidor**
    
    - **Gestión de Backups:** Restauración de la base de datos ante desastres utilizando los archivos `.tar.gz`.
    - **Monitorización y Logs:** Inspección de registros del servidor en tiempo real para depuración.
    - **Reinicio de Servicios Seguro:** Comandos de Docker para recargar Synapse sin corromper la base de datos.
    - **Actualización del Certificado CSIRC:** Sustitución del archivo XML caducado para restablecer el SSO.
    - **Actualización de la Interfaz Web:** Aplicación de cambios estéticos en caliente sin reiniciar el backend.
    - **Gestión del Media Store:** Estructura de archivos adjuntos (hashes) y su legibilidad según el cifrado.
    
- **5.2. Administración de la Aplicación**
    
    - **Acceso a Synapse-Admin:** Pasos para entrar y navegar por el panel de control gráfico.
	- **Acceso directo a la Base de datos (SQLite):** Como entrar e interactuar con la base de datos SQlite a través de la terminal
    - **Gestión de Administradores:** Promoción de usuarios estándar a administradores del sistema.
    - **Desactivación de Usuarios:** Proceso de baneo por credenciales y revocación de sesiones.
    - **Privacidad y Visibilidad de Salas:** Diferencias operativas entre salas privadas, públicas e invitaciones.
    - **Permisos de Directorio:** Inclusión manual de administradores en el `homeserver.yaml` para publicar salas públicas.
    - **Sincronización de Dispositivos:** Manual de usuario para verificar sesiones nuevas (E2EE) y desencriptar mensajes
