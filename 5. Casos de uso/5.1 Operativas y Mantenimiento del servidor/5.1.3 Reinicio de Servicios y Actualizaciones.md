
En este caso de uso se describe el procedimiento para aplicar cambios de configuración o actualizaciones reiniciando los contenedores Docker, teniendo cuidado en ejecutar los comandos correctos dependiendo del nivel del cambio para evitar problemas en la base de datos.

1. **Requisitos previos:**
	
	- Haber guardado los cambios en el archivo de configuración correspondiente
	- Tener acceso SSH con permisos de sudo a la maquina.
	- Situarnos en el directorio `cd /home/svc`
	

2. **Procedicimiento:**

- **a) Para cambios en archivos de configuración (reinicio rápido)**

SI hemos modificado un archivo estático como puede ser el `data/homeserver.yaml` o `element-config.json` necesitamos un reinicio dependiendo de si estamos trabajando por un lado con el synapse (backend) o el element (frontend) 

``` Bash
# Si modificamos homeserver.yaml o ugr-idp-metadata.xml
sudo docker compose restart synapse

# Si modificamos element-config.json
sudo docker compose restart element
```

**Nota:** Si no sabes cual de los dos es el que tienes que ejecutar, pues ejecuta los dos tampoco hay ningún problema.

- **b) Para cambios en `docker-compose.yml` (reinicio profundo)**

SI hemos editado el docker-compose o queremos actualizar las imágenes de synapse o element a la última versión debemos apagar y encender el servicio por completo.

```Bash
sudo docker compose down
sudo docker compose up -d
```
