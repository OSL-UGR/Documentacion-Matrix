
En este caso de uso se describe el procedimiento para sustituir el archivo de XML de metadatos si hubiese cambiado o se hubiese modificado del Proveedor de Identidad de la UGR (IdP). Es necesario actualizarlo para restaurar el inicio de sesión oficial (SSO) y permitir así el acceso al servidor Matrix.

1. **Requisitos previos:**
	
	- Disponer del nuevo archivo XML proporcionado por el CSIRC.
	- Tener acceso SSH con permisos de sudo a la maquina.
	- Situarnos en el directorio `cd /home/svc`
	

2. **Procedimiento:**

- **a) Reemplazar el archivo:**

Tenemos que copiar o mover el nuevo certificado al directorio de datos asegurándonos de que se sobrescribiendo al antiguo manteniendo exactamente el mismo nombre que es el que espera el archivo de configuración `homeserver.yaml`.


``` bash
sudo cp /directorio/nuevo-metadata.xml ./data/ugr-idp-metadata.xml
```

- **b) Reinicio del servicio:**

Para que Synapse lea el nuevo certificado es necesario realizar un reinicio del contenedor:

``` bash
sudo docker compose restart synapse
```
