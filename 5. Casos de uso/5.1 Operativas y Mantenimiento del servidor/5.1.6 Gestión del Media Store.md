
En este caso de uso se describe el procedimiento para explorar el directorio donde se almacenan los archivos multimedia (imágenes, vídeos, documentos), entendiendo su estructura de carpeta basada en un hash y verificando que tipos de archivos se encuentras con el cifrado E2EE y cuales no.

1. **Requisitos previos:**
	
	- Tener acceso SSH con permisos de sudo a la maquina.
	- Situarnos en el directorio `cd /home/svc`
	
2. **Procedimiento:**

- **a) Donde se guardan las imágenes en la base de datos:**

Lo primero que tenemos que tener en cuenta, es que los archivos pesados (multimedia) no se guardan dentro del `data/homeserver.db`. En la base de datos solo existe un registro que apunta a la ubicación real de los archivos que se encuentra en la carpeta del Media Store. Nos vamos a mover a esa carpeta

```bash
cd data/media_store/
```

- **b) Inspeccionamos el directorio:**

Si listamos su contenido con el comando `tree -L 2` por ejemplo no encontraremos los archivos jpg ni pdf directamente, si no un subsistema de carpetas que se divide en 3 subcarpetas principales, y a su vez dentro de estas se ordenan por rangos alfabéticos, para aumentar la eficiencia de consulta al estilo tabla hash. En estas 3 subcarpetas encontraremos lo siguiente:

- **local_content**: Archivos subidos por los usuarios del servidor Matrix.

- **remote_content**: Caché de archivos de otros servidores externos (ej. matrix.org) descargados para no consumir ancho de banda repetidas veces.

- **thumbnails**: Miniaturas generadas automáticamente para las previsualizaciones.

- **c) Como inspeccionar los archivos sin cifrar (pertenecientes a salas públicas)**

Si el archivo fue enviado a una sala pública sin cifrar o es una foto de perfil, el administrador puede ver su información y visualizarlo ya que esta se encuentra en texto plano. Por ejemplo podemos comprobar su tipo de archivo ejecutando:

``` bash
file local_content/al/ar/nombre-raro-del-archivo-xd
```


- **d) Como se almacenan los archivos cifrados (salas con E2EE activado)**

Si el archivo fue enviado en una sala privada con el Cifrado De Extremo a Extremo activado, el cliente Element del usuario lo encriptó antes de subirlo, por lo tanto es totalmente inaccesible. Si intentamos repetir el paso anterior o abrir el archivo solo veremos datos ininteligibles. Es matemáticamente imposible para el administrador descifrar este contenido, garantizando así la privacidad total de los usuarios.
