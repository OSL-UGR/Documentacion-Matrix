
En este caso de uso se describe el procedimiento para aplicar cambios estéticos o cambios en el comportamiento del cliente web modificando su archivo de configuración. En este punto cabe recalcar que no va a ser necesario reiniciar el servidor backend ni interrumpir las conexiones activas de los usuarios.

1. **Requisitos previos:**
	
	- Revisar y conocer la  [documentación oficial de Element](https://github.com/element-hq/element-web/blob/develop/docs/config.md) para ver que parámetros de configuración queremos modificar o añadir.
	- Tener acceso SSH con permisos de sudo a la maquina.
	- Situarnos en el directorio `cd /home/svc`
	

2. **Procedimiento:**

- **a) Editamos el archivo de configuración:**

Abrimos el achivo JSON responsable del fronted con nuestro editor de texto favorito en la terminal:

``` bash
sudo nano element-config.json
```

**Nota:** Es muy importante comprobar que el json que dejamos en el archivo se encuentre bien estructurado y pareado, si no puede dar fallos y no cargar el fronted de nuestro servidor. Para comprobarlo es recomendable utilizar algún tipo de comprobador web como el siguiente:

- [Comprobador web de Json](https://www.site24x7.com/es/tools/json-formateador.html)

- **b) Aplicar los cambios:**

No se nos puede olvidar reiniciar el contenedor de Element (fronted) para que los cambios que hemos realizado se apliquen automáticamente una vez se actualice (Ctrl+F5) la página oficial desde el navegador:

``` bash
sudo docker compose restart element
```
