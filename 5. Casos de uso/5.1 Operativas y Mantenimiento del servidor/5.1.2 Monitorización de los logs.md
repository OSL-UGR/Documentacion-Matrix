
En este caso de uso se describe el procedimiento para monitorizar los logs con el objetivo de diagnosticar problemas de acceso o errores internos del servidor de Docker. 

1. **Requisitos previos:**
	
	- Tener acceso SSH con permisos de sudo a la maquina.
	- Situarnos en el directorio `cd /home/svc`
	
2. **Procedimiento:** 

Podemos visualizarlos de dos maneras posibles:

- **a) En tiempo real (de forma dinámica):**

De esta manera dejamos la terminal abierta y podemos ver en tiempo real el flujo de mensaje de logs según van llegando, es decir, cada vez que un usuario inicia sesión, que se envía algún mensaje, se realiza alguna petición etc. 

```Bash
sudo docker compose logs -f synapse
```


- **b) Unas líneas específicas (de forma estática):**

Por si queremos hacer una consulta rápida de forma estática sin que se vaya actualizando constantemente con los nuevos mensajes que vayan entrando. Con el siguiente comando se imprimen las últimas 50 líneas, si cambiamos ese valor podemos calibrar este valor al gusto:

``` Bash
sudo docker compose logs --tail 50 synapse
```
