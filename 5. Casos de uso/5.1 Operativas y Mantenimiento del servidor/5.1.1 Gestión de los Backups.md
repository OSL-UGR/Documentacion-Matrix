
En este caso de uso se describe el procedimiento para recuperar el estado del servidor a partir de alguna de las copias comprimida y alojadas en `/home/svc/backups/`. Este proceso es necesario en caso de corrupción de la base de datos o de pérdida de archivos claves de configuración.

1. **Requisitos previos:**
	
	- Tener localizado el archivo de backup deseado (ej: `backup_synapse_2026-05-02_03-00.tar.gz`)
	- Tener acceso SSH con permisos de sudo a la maquina.
	- Situarnos en el directorio `cd /home/svc`
	
2. **Procedimiento:**

- **a) Tenemos que parar los servicios:**

No podemos sobreescribir archivos mientras los contenedores están activos:

``` Bash
sudo docker compose stop
```

- **b) Extraemos el backup deseado:**

Restauramos el contenido del bakup directamente sobre el directorio actual. Es importante poner el `-C ./` para asegurar que los archivos extraídos caigan en sus rutas originales.

``` Bash
sudo tar -xzvf ./backups/NOMBRE_BACKUP.tar.gz -C ./
```

- **c) Reiniciamos el servicio:**

Una vez los archivos han sido sustituidos por las versiones de la copa de seugirdad, levantamos de vuelta el servicio:

``` Bash
sudo docker compose start synapse
```
