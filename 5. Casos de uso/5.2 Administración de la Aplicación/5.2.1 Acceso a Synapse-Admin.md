
En este caso de uso se describe el procedimiento para acceder e iniciar sesión en Synapse-Admin, que es la interfaz gráfica que se ha configurado para poder gestionar la base de datos del Matrix. De esta manera podemos acceder a usuarios, salas y permisos de forma mucho más visual sin necesidad de interactuar directamente con la base de datos SQlite por terminal.

1. **Requisitos previos:**
	
	- Tener acceso a una cuenta con permisos de admin en el servidor Matrix.
	

2. **Procedimiento:**

- **a) Obtener la IP del servidor matrix:**

Esto es tan sencillo como realizar un ping al dominio oficial del servidor alojado en `matrix.ugr.es` y obtener su dirección IP.

``` Bash
ping matrix.ugr.es
```

**Nota:** Ejecutar el comando para saldar de dudas, pero si no se ha cambiado su dirección lógica a fecha en la que se escribe esta documentación, el valor de la IP es: `150.214.104.41`

- **b) Acceso a la interfaz:**

Tenemos que abrir un navegador y tenemos que introducir la siguiente url IMPORTANTE que sea con `http` y no con `https` apuntando al puerto `8081` configurado como administración:

``` Navegador
http://150.214.104.41:8081
```

- **c) Inicio de sesión con SSO:**

En la pantalla de inicio de sesión de Synapse-Admin no debemos introducir ni el usuario ni la contraseña. En su lugar en el campo de `Homeserver URL` tenemos que añadir el enlace para que apunte a la IP interna de la API de Synapse en el puerto `8008` y así acceder al inicio de sesión oficial de la universidad. El enlace es el siguiente:

``` Homeserver URL
http://150.214.104.41:8008
```

**Nota:** Realmente lo de la IP exacta no debería de ser necesaria, con poner el nombre del dominio `matrix-ugr.es` debería ser suficiente. Sin embargo una vez accediendo con el dominio no me dejaba acceder por no se que error, así que desde entonces accedo siempre con la IP y nunca me ha dado ningún fallo.
