
En este caso de uso se describe el procedimiento para autorizar manualmente a un administrador a que publique salas en el directorio raíz, limitando esta funcionalidad a unos usuarios específicos.

1. **Requisitos previos:**

	- Tener acceso SSH con permisos de sudo a la maquina.
	- Situarnos en el directorio `cd /home/svc`
	
2. **Procedimiento:**

Por medidas de seguridad ningún usuario tiene permisos para publicar salas en el directorio por defecto. Así que para añadirlos debemos añadirlos a mano dentro del homeserver:

**a) Abrimos el homeserver.yaml para editarlo:**

``` bash
sudo nano data/homeserver.yaml
```

**b) Introducimos el usuario:**  Tenemos que ir hasta el final del documento donde se encuentra el bloque con nombre `room_list_publication_rules`. Dentro de este añadiremos el identificador de nuestro nuevo administrador y el resto de campos de la siguiente manera.


``` nano
  - user_id: "@nuevo_admin:matrix.ugr.es"
    alias: "*"
    room_id: "*"
    action: allow
```

_(Guardamos los cambios con `Ctrl+O` y cerramos el editor con `Ctrl+X`)._

**c) Reiniciamos el backend Synapse para aplicar los cambios**

``` bash
sudo docker compose restart synapse
```

A partir de este momento, dicho usuario podrá usar el botón de "Publicar sala" para modificar la visibilidad de esta.
