
Esta sección documenta la gestión de los nombres públicos y legibles de las salas. Dado que Matrix genera identificadores internos alfanuméricos incomprensibles (ej. `!aBcDeF:ugr.es`), los alias permiten crear "enlaces amigables" (ej. `#ia-grupo-a:ugr.es`) que pueden publicarse directamente en PRADO para que los estudiantes hagan clic y accedan al canal de la asignatura.

---

## Resolver Alias (Obtener ID de Sala)
**Endpoint:** `[GET] /_matrix/client/v3/directory/room/{roomAlias}`

> **Propósito:** Consulta al servidor para convertir un alias legible en el ID interno real de la sala a la que apunta.

### Aplicación en el Proyecto PRADO
Paso de validación y búsqueda. Antes de que el panel intente crear un nuevo alias para una asignatura, consultará este endpoint para comprobar si el nombre ya está en uso. También sirve para que el script de sincronización recupere el ID interno de una sala a partir de una URL compartida en Moodle.

### Parámetros Clave (Ruta)
* `roomAlias`: (String) El alias completo a buscar (ej. `#inteligencia-artificial:ugr.es`).

### Notas de Implementación / Seguridad
No requiere autenticación. Si el alias pertenece a otro dominio, el servidor intentará resolverlo por federación. Devuelve un objeto JSON con el `room_id` correspondiente y un array `servers` con los nodos conscientes de dicha sala.

## Crear un Nuevo Alias para una Sala
**Endpoint:** `[PUT] /_matrix/client/v3/directory/room/{roomAlias}`

> **Propósito:** Crea un nuevo mapeo en el directorio del servidor, vinculando un alias público a un ID de sala interno.

### Aplicación en el Proyecto PRADO
Acción fundamental tras el aprovisionamiento. Una vez que el script (basado en los datos de Moodle) crea el "Espacio" o la sala para una asignatura, invocará este endpoint para asignarle su identificador institucional oficial, haciéndola accesible mediante un enlace estandarizado.

### Parámetros Clave (Ruta y Payload)
* `roomAlias`: (String) [Ruta] El alias que se desea registrar.
* `room_id`: (String) [Payload] El ID interno de la sala a la que apuntará el alias.

### Notas de Implementación / Seguridad
Requiere autenticación mediante Token de acceso. El servidor local solo permite crear alias bajo su propio dominio.

---

## Eliminar un Alias de Sala
**Endpoint:** `[DELETE] /_matrix/client/v3/directory/room/{roomAlias}`

> **Propósito:** Destruye el mapeo entre un alias y un ID de sala en el directorio del servidor.

### Aplicación en el Proyecto PRADO
Limpieza y mantenimiento. Si una asignatura cambia de código en el plan de estudios, o si se cierra el entorno virtual de un año académico anterior, el panel de administración utilizará este endpoint para liberar el alias. De este modo, el enlace antiguo dejará de funcionar y el alias podrá reciclarse para el nuevo curso.

### Parámetros Clave (Ruta)
* `roomAlias`: (String) El alias que se desea eliminar.

### Notas de Implementación / Seguridad
Requiere autenticación. Los servidores aplican comprobaciones estrictas de control de acceso: por norma general, un alias solo puede ser eliminado por el usuario que lo creó o por un administrador global del servidor Synapse.

---

## Listar Alias Locales de una Sala
**Endpoint:** `[GET] /_matrix/client/v3/rooms/{roomId}/aliases`

> **Propósito:** Devuelve una lista de todos los alias que el servidor local mantiene activos y que apuntan a una sala específica.

### Aplicación en el Proyecto PRADO
Tareas de auditoría estructural. Permite al panel de administración comprobar cuántos enlaces de acceso directos existen actualmente para una misma asignatura (por si algún profesor ha creado alias manuales secundarios) y listarlos para su revisión.

### Parámetros Clave (Ruta)
* `roomId`: (String) El ID interno de la sala sobre la que se quiere consultar.

### Notas de Implementación / Seguridad
Requiere autenticación. La API especifica que el usuario que hace la llamada debe ser miembro de la sala, a menos que el historial de la sala esté configurado como `world_readable` (público total) o se realice mediante credenciales de administrador del servidor.
