

Esta sección define cómo interactuar con la información pública de las cuentas de Matrix. En el contexto de la integración, es la herramienta que permite abandonar los identificadores técnicos abstractos (como `@x12345:ugr.es`) y mostrar la identidad real y visual del alumno o profesor extraída del campus virtual.

---

## Obtener el Perfil Completo del Usuario
**Endpoint:** `[GET] /_matrix/client/v3/profile/{userId}`

> **Propósito:** Recupera toda la información pública del perfil de un usuario, incluyendo su nombre a mostrar, URL de avatar y zona horaria.

### Aplicación en el Proyecto PRADO
El panel de administración utilizará este endpoint durante sus tareas de auditoría para verificar el estado de una cuenta. Permite comprobar de un solo vistazo si un alumno ya tiene un nombre real asignado o si su perfil está vacío y requiere una sincronización de datos desde Moodle.

### Parámetros Clave (Ruta)
* `userId`: (String) El Matrix ID completo del usuario cuyo perfil se desea consultar.

### Notas de Implementación / Seguridad
Por lo general, no requiere autenticación para consultar perfiles de usuarios dentro del mismo servidor, aunque esto puede variar según las políticas de federación y privacidad configuradas en Synapse. Devuelve un objeto JSON con claves como `displayname` y `avatar_url`.

---

## Obtener un Campo Específico del Perfil
**Endpoint:** `[GET] /_matrix/client/v3/profile/{userId}/{keyName}`

> **Propósito:** Recupera el valor de un único campo concreto del perfil del usuario, minimizando el tamaño de la respuesta.

### Aplicación en el Proyecto PRADO
Útil para validaciones ligeras. Si el script de sincronización solo necesita comprobar si la foto de perfil ha cambiado respecto a la base de datos de PRADO, puede consultar exclusivamente la clave del avatar sin descargar el resto de información.

### Parámetros Clave (Ruta)
* `userId`: (String) El Matrix ID del usuario.
* `keyName`: (String) El nombre del campo a consultar. Valores estándar: `displayname` (nombre), `avatar_url` (foto de perfil) o `m.tz` (zona horaria).

### Notas de Implementación / Seguridad
No requiere autenticación. El servidor devuelve un JSON que contiene únicamente la clave solicitada y su valor actual.

---

## Actualizar un Campo del Perfil
**Endpoint:** `[PUT] /_matrix/client/v3/profile/{userId}/{keyName}`

> **Propósito:** Establece o actualiza el valor de un campo específico en el perfil del usuario.

### Aplicación en el Proyecto PRADO
**El endpoint más importante de la sección.** El script de sincronización ejecutará este comando en dos fases tras crear la cuenta del alumno:
1. Llamará al endpoint con `keyName = displayname` para inyectar el nombre real ("Nombre Apellidos").
2. Llamará al endpoint con `keyName = avatar_url` para vincular la foto de perfil oficial del estudiante.

### Parámetros Clave (Ruta y Payload)
* `userId`: (String) [Ruta] El Matrix ID del usuario.
* `keyName`: (String) [Ruta] La clave que se va a actualizar (`displayname` o `avatar_url`).
* El cuerpo de la petición (Payload) debe ser un objeto JSON con una única propiedad que coincida exactamente con el `keyName` especificado en la URL. El valor asociado será la cadena de texto con el nombre o la URI de la imagen.

### Notas de Implementación / Seguridad
Requiere autenticación mediante Token de acceso. El cliente debe tener permisos para modificar la cuenta. La documentación técnica establece que el tamaño total del perfil no debe superar los 64 KiB. Para el `avatar_url`, Matrix requiere que la imagen ya esté alojada en el servidor interno y se proporcione como una URI con formato `mxc://`.

---

## Eliminar un Campo del Perfil
**Endpoint:** `[DELETE] /_matrix/client/v3/profile/{userId}/{keyName}`

> **Propósito:** Elimina de forma definitiva un campo específico del perfil del usuario y su valor asociado.

### Aplicación en el Proyecto PRADO
Garantiza la consistencia de datos a la inversa. Si un estudiante o profesor decide eliminar su foto de perfil en el campus virtual de Moodle, el script de sincronización detectará este cambio y utilizará este endpoint para purgar el avatar en Matrix, evitando mostrar información obsoleta.

### Parámetros Clave (Ruta)
* `userId`: (String) El Matrix ID del usuario.
* `keyName`: (String) La clave del campo que se desea eliminar (ej. `avatar_url`).

### Notas de Implementación / Seguridad
Requiere autenticación con Token de acceso. La API recomienda expresamente utilizar este método DELETE en lugar de enviar valores nulos mediante el método PUT cuando la intención es borrar completamente la información.
