# 7.5 Gestión de Sesiones (Session Management)


## Obtener Tipos de Inicio de Sesión Soportados
**Endpoint:** `[GET] /_matrix/client/v3/login`

> **Propósito:** Consulta al servidor local cuáles son los mecanismos de autenticación que tiene habilitados para permitir el acceso a los usuarios.

### Aplicación en el Proyecto PRADO
Útil en la fase de inicialización del puente. El script de conexión interrogará al servidor para confirmar si la autenticación por contraseña tradicional (`m.login.password`) está activa para las cuentas locales de los bots o administradores, o si requiere otros flujos de autenticación.

### Parámetros Clave
* No requiere parámetros de ruta ni de consulta (Query).

### Notas de Implementación / Seguridad
No requiere autenticación previa. Devuelve un objeto JSON con un array `flows`, donde cada elemento indica un `type` de inicio de sesión soportado (ej. `m.login.password`, `m.login.token`).

---

## Autenticar al Usuario (Login)
**Endpoint:** `[POST] /_matrix/client/v3/login`

> **Propósito:** Autentica a un usuario y emite un token de acceso que podrá utilizarse para autorizar peticiones subsecuentes en la API. Además, registra un dispositivo para esa sesión.

### Aplicación en el Proyecto PRADO
El motor de arranque de la automatización. Tus scripts programados (como el de matriculación nocturna o los bots) utilizarán este endpoint proporcionando sus credenciales para obtener el Token de Acceso (`access_token`). Es fundamental que el panel guarde este token y lo reutilice para las operaciones, en lugar de realizar un login nuevo por cada alumno matriculado (lo que generaría cientos de dispositivos huérfanos).

### Parámetros Clave (Payload)
* `type`: (String) El tipo de autenticación utilizado (obligatorio, debe coincidir con uno de los devueltos en el endpoint GET `/login`, típicamente `m.login.password`).
* `identifier`: (Object) Objeto de identificación moderna del usuario. Requiere un subcampo `type` (ej. `m.id.user`) y el identificador correspondiente (ej. `user: "bot_prado"`). Sustituye a los campos obsoletos directos como `user` o `address`.
* `password`: (String) La contraseña de la cuenta (obligatoria si el tipo es `m.login.password`).
* `device_id`: (String) ID del dispositivo del cliente. Si no se proporciona, el servidor autogenerará uno.
* `initial_device_display_name`: (String) Nombre descriptivo para el nuevo dispositivo (ej. "Script-Sincronizacion-PRADO").

### Notas de Implementación / Seguridad
No requiere autenticación previa. La respuesta JSON contiene campos vitales: `access_token`, `device_id` y `user_id`. El token devuelto queda asociado intrínsecamente al dispositivo registrado.

## Cerrar Sesión (Invalidar Token Actual)
**Endpoint:** `[POST] /_matrix/client/v3/logout`

> **Propósito:** Invalida el token de acceso existente utilizado en la petición, impidiendo su uso futuro para autorización. Elimina simultáneamente el dispositivo asociado y sus claves criptográficas.

### Aplicación en el Proyecto PRADO
Buenas prácticas de seguridad y limpieza. Si el panel de administración inicia una sesión temporal específica para una tarea de auditoría, debe llamar a este endpoint al finalizar la ejecución para asegurarse de que el token no queda activo en la base de datos de Matrix.

### Parámetros Clave
* No requiere parámetros en el cuerpo de la petición. El token a invalidar se extrae de la cabecera HTTP de autorización (`Authorization: Bearer <token>`).

### Notas de Implementación / Seguridad
Requiere autenticación. Tras devolver un código HTTP 200, cualquier llamada posterior con ese mismo token devolverá un error.

---

## Cerrar Todas las Sesiones (Logout All)
**Endpoint:** `[POST] /_matrix/client/v3/logout/all`

> **Propósito:** Invalida todos los tokens de acceso activos asociados a la cuenta del usuario de forma global. Elimina todos los dispositivos registrados y sus respectivas claves.

### Aplicación en el Proyecto PRADO
El "botón de emergencia". Si desde el servicio de informática de la UGR se detecta que la cuenta de un profesor, un delegado o uno de los propios bots ha sido comprometida, el panel de administración puede disparar este endpoint (usando el token actual del bot administrador suplantando al usuario o el token comprometido si se conoce) para expulsar al atacante inmediatamente de todas las sesiones activas simultáneamente.

### Parámetros Clave
* No requiere parámetros en el cuerpo de la petición.

### Notas de Implementación / Seguridad
Requiere autenticación. La documentación especifica que este endpoint *no* utiliza la API de Autenticación Interactiva del Usuario (UIA) a propósito, para evitar que un atacante que ha robado una sesión necesite la contraseña para expulsar a los demás. Cierra el acceso total de inmediato.
