
Esta sección abarca las herramientas fundamentales para darle vida a las salas. Permite el envío y lectura de mensajes (el núcleo de cualquier bot), la modificación de la configuración estructural de la sala (roles, permisos y cifrado) y la moderación de contenidos. Es indispensable para la automatización de la docencia y la interacción en tiempo real.

## Enviar un Evento de Mensaje

**Endpoint:** `[PUT] /_matrix/client/v3/rooms/{roomId}/send/{eventType}/{txnId}`

> **Propósito:** Envía un evento a la línea de tiempo de la sala, utilizado principalmente para mensajes puntuales de chat (texto, imágenes o archivos).

### Aplicación en el Proyecto PRADO

Es el motor principal para las notificaciones y los bots. El panel de administración o un bot de secretaría utilizarán este endpoint para enviar recordatorios de entrega de prácticas, avisos del profesor o respuestas automáticas a dudas recurrentes de los alumnos.

### Parámetros Clave (Ruta y Payload)

- `roomId`: (String) [Ruta] El ID interno de la sala.
    
- `eventType`: (String) [Ruta] El tipo de evento (generalmente `m.room.message`).
    
- `txnId`: (String) [Ruta] ID de transacción generado por el cliente. El servidor lo usa para garantizar la idempotencia (si la red falla y se reenvía la petición, el servidor no duplicará el mensaje).
    
- El payload (cuerpo) contiene los datos del mensaje, que varían según el `eventType` (ej. el cuerpo del texto).
    

### Notas de Implementación / Seguridad

Requiere autenticación. El nivel de poder (Power Level) del usuario remitente debe ser igual o superior al requerido para enviar mensajes en esa sala específica.

## Enviar un Evento de Estado

**Endpoint:** `[PUT] /_matrix/client/v3/rooms/{roomId}/state/{eventType}/{stateKey}`

> **Propósito:** Modifica o establece una configuración persistente en la sala. Si un evento con el mismo tipo y clave ya existe, se sobrescribe.

### Aplicación en el Proyecto PRADO

Es la herramienta maestra de configuración. Se utilizará para tres funciones críticas:

1. Actualizar el nombre o descripción de la asignatura en tiempo real (`m.room.name`).
    
2. Modificar los niveles de poder (`m.room.power_levels`) para otorgar permisos de Administrador a los profesores y Moderador a los delegados de clase.
    
3. Desactivar el cifrado de extremo a extremo (enviando la configuración pertinente) en canales públicos de teoría donde prime la auditoría frente a la privacidad estricta.
    

### Parámetros Clave (Ruta y Payload)

- `roomId`: (String) [Ruta] El ID interno de la sala.
    
- `eventType`: (String) [Ruta] El tipo de configuración a alterar.
    
- `stateKey`: (String) [Ruta] Identificador del estado. Para configuraciones globales de sala se deja en blanco. Para estados aplicados a un usuario concreto (como asignar un rol), se especifica aquí su Matrix ID.
    

### Notas de Implementación / Seguridad

Requiere un Power Level alto (generalmente administrador). A diferencia del envío de mensajes, este endpoint no utiliza ID de transacción (`txnId`).

## Obtener Todo el Estado Actual de una Sala

**Endpoint:** `[GET] /_matrix/client/v3/rooms/{roomId}/state`

> **Propósito:** Recupera la lista completa de eventos de estado vigentes en la sala, proporcionando una fotografía exacta de su configuración actual.

### Aplicación en el Proyecto PRADO

Útil para auditorías previas de configuración. Antes de que el panel modifique un permiso o aplique un cambio masivo a las salas de un grado, consultará este endpoint para verificar cómo está configurada la sala (ej. comprobar si un profesor ya tiene nivel de administrador o descargar la configuración actual de permisos antes de editarla).

### Parámetros Clave (Ruta)

- `roomId`: (String) [Ruta] El ID interno de la sala.
    

### Notas de Implementación / Seguridad

Requiere autenticación. El usuario debe formar parte de la sala (o haber pertenecido a ella si su visibilidad histórica lo permite).

## Sincronizar Estado y Recibir Mensajes (Sync)

**Endpoint:** `[GET] /_matrix/client/v3/sync`

> **Propósito:** Mantiene sincronizado al cliente con el servidor, obteniendo un delta de nuevos eventos, mensajes y cambios ocurridos desde la última consulta.

### Aplicación en el Proyecto PRADO

Es el "oído" de cualquier bot interactivo de la universidad. Un script (demonio) realizará llamadas continuas a este endpoint (técnica de long-polling) para capturar en tiempo real los mensajes que escriben los estudiantes y permitir que el FAQBot detecte comandos o responda preguntas del temario de forma instantánea.

### Parámetros Clave (Query)

- `since`: (String) Token de lote (`next_batch`) devuelto en la sincronización anterior. Garantiza que solo se descarguen mensajes nuevos.
    
- `timeout`: (Integer) Tiempo máximo de espera en milisegundos. Permite mantener la conexión abierta hasta que llegue un mensaje nuevo, reduciendo la carga del servidor.
    
- `filter`: (String) ID o JSON de un filtro para descartar eventos innecesarios (ej. escuchar solo mensajes de texto y omitir eventos de escritura).
    
- `set_presence`: (Enum) Permite ocultar el estado de conexión del bot (`offline`).
    

### Notas de Implementación / Seguridad

Las respuestas contienen un token `next_batch` obligatorio para la siguiente petición. Es uno de los endpoints más pesados, por lo que el uso estricto de filtros (`filter`) es altamente recomendable en bots de servidor.

## Listar Usuarios Unidos Actualmente

**Endpoint:** `[GET] /_matrix/client/v3/rooms/{roomId}/joined_members`

> **Propósito:** Devuelve un mapa de los Matrix IDs de todos los usuarios que están actualmente unidos a la sala, junto con los datos básicos de su perfil.

### Aplicación en el Proyecto PRADO

Ideal para realizar auditorías rápidas de asistencia o control de acceso. El panel ejecutará este endpoint para extraer una lista limpia de quién está en el canal y contrastarla instantáneamente con la base de datos de matriculados en Moodle, sin necesidad de procesar todo el historial de eventos de la sala.

### Parámetros Clave (Ruta)

- `roomId`: (String) [Ruta] El ID interno de la sala.
    

### Notas de Implementación / Seguridad

Es significativamente más rápido y eficiente en consumo de recursos a nivel de servidor que consultar todos los eventos de membresía individuales de la sala.

## Redactar (Eliminar) un Evento

**Endpoint:** `[PUT] /_matrix/client/v3/rooms/{roomId}/redact/{eventId}/{txnId}`

> **Propósito:** Elimina permanentemente la información crítica de un evento en el servidor (borrado de mensajes), de forma irreversible.

### Aplicación en el Proyecto PRADO

Herramienta clave para la moderación automatizada y manual. Permite a los profesores, o a un bot con permisos de moderación, borrar de inmediato mensajes inapropiados, spam o respuestas correctas a ejercicios compartidas indebidamente en el canal general de la clase.

### Parámetros Clave (Ruta y Payload)

- `roomId`: (String) [Ruta] El ID interno de la sala.
    
- `eventId`: (String) [Ruta] El ID exacto del mensaje que se va a eliminar.
    
- `txnId`: (String) [Ruta] ID de transacción único.
    
- `reason`: (String) [Payload] Motivo opcional del borrado que quedará registrado (ej. "Infracción de normas de convivencia").
    

### Notas de Implementación / Seguridad

Requiere autenticación. El usuario debe tener un Power Level superior o igual al nivel de redacción definido en la configuración de la sala. El borrado es definitivo y se propaga a todos los clientes.
