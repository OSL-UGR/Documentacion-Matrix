
Esta sección engloba las herramientas necesarias para gestionar el ciclo de vida de los usuarios dentro de las salas y espacios. Es la piedra angular de la sincronización en tiempo real: permite forzar la entrada de estudiantes cuando se matriculan, expulsarlos cuando se dan de baja, o restringir su acceso en caso de sanciones disciplinarias.

---

## Invitar a un Usuario a una Sala
**Endpoint:** `[POST] /_matrix/client/v3/rooms/{roomId}/invite`

> **Propósito:** Envía una invitación oficial a un usuario para que se una a una sala específica. Genera un evento `m.room.member` con estado `invite`.

### Aplicación en el Proyecto PRADO
Paso 1 del flujo de alta. Cuando el panel detecte una nueva matrícula en Moodle, el "bot" o cuenta administradora de la asignatura utilizará este endpoint para invitar al MXID del alumno a la sala correspondiente.

### Parámetros Clave (Payload y Ruta)
* `roomId`: (String) [Ruta] El ID interno de la sala (`!xxx:ugr.es`). No admite alias.
* `user_id`: (String) [Payload] El Matrix ID completo del alumno a invitar (ej: `@pepe:ugr.es`).
* `reason`: (String) [Payload] Razón opcional (ej: "Matriculación automática en PRADO").

### Notas de Implementación / Seguridad
Requiere autenticación mediante Token de acceso. Solo los usuarios que ya están en la sala y tienen el "power level" adecuado pueden invitar a otros. El usuario no participa en la sala hasta que no acepta la invitación (paso siguiente).

## Forzar Unión (Join) a una Sala
**Endpoint:** `[POST] /_matrix/client/v3/join/{roomIdOrAlias}` *(También aplicable a `/rooms/{roomId}/join`)*

> **Propósito:** Introduce a un usuario en una sala (cambiando su estado a `join`), permitiéndole leer y escribir eventos.

### Aplicación en el Proyecto PRADO
Paso 2 del flujo de alta. Para evitar que los alumnos tengan decenas de invitaciones pendientes, el panel utilizará la Admin API de Synapse para ejecutar este comando *haciéndose pasar* por el alumno, aceptando automáticamente la invitación del paso anterior.

### Parámetros Clave (Payload y Ruta)
* `roomIdOrAlias`: (String) [Ruta] Permite unirse tanto por ID interno como por alias (`#ia:ugr.es`).
* `reason`: (String) [Payload] Razón opcional que quedará registrada.
* `third_party_signed`: (Object) [Payload] Solo necesario si la sala exige firmas de servidores de identidad. No se usará en este proyecto.

### Notas de Implementación / Seguridad
Si la sala es privada (lo habitual en asignaturas), el usuario *debe* haber sido invitado previamente (paso 1). Si la sala es pública, puede unirse directamente.

## Listar las Salas Actuales de un Usuario
**Endpoint:** `[GET] /_matrix/client/v3/joined_rooms`

> **Propósito:** Devuelve una lista de los IDs de todas las salas a las que el usuario autenticado pertenece actualmente.

### Aplicación en el Proyecto PRADO
Vital para el script de auditoría nocturna. El sistema puede comparar la lista de salas Matrix en las que está un alumno contra su lista de asignaturas matriculadas en la base de datos de PRADO. Si hay discrepancias (está en una sala pero no en la BD, o viceversa), el script lanza correcciones automáticas.

### Parámetros Clave
* No requiere parámetros.

### Notas de Implementación / Seguridad
Requiere autenticación. Solo devuelve los IDs de las salas (`room_id`), no los metadatos completos.

## Expulsar (Kick) a un Usuario
**Endpoint:** `[POST] /_matrix/client/v3/rooms/{roomId}/kick`

> **Propósito:** Expulsa a un usuario de la sala, cambiando su estado de membresía a `leave`. El usuario podrá volver a entrar si es invitado de nuevo.

### Aplicación en el Proyecto PRADO
Se ejecutará automáticamente si un alumno anula su matrícula, finaliza el curso o si su cuenta caduca en PRADO, garantizando que pierde el acceso a la información y comunicaciones privadas de la asignatura.

### Parámetros Clave (Payload y Ruta)
* `roomId`: (String) [Ruta] El ID interno de la sala.
* `user_id`: (String) [Payload] El MXID del usuario a expulsar.
* `reason`: (String) [Payload] Razón de la expulsión (ej: "Baja en matrícula oficial").

### Notas de Implementación / Seguridad
Requiere autenticación. El usuario que hace la llamada (el bot del panel) debe tener un "power level" superior o igual al requerido para expulsar (`kick`) definido en los ajustes de la sala, y además superior al del objetivo.

## Abandonar (Leave) una Sala
**Endpoint:** `[POST] /_matrix/client/v3/rooms/{roomId}/leave`

> **Propósito:** El usuario decide voluntariamente dejar de participar en una sala o rechazar una invitación pendiente.

### Aplicación en el Proyecto PRADO
A nivel de sistema, es idéntico al `Kick`, pero lo ejecuta el propio usuario. En un entorno académico estricto, es posible que PRADO fuerce a los alumnos a re-unirse automáticamente si intentan salirse de un grupo obligatorio.

### Parámetros Clave (Payload y Ruta)
* `roomId`: (String) [Ruta] El ID interno de la sala.
* `reason`: (String) [Payload] Razón opcional.

### Notas de Implementación / Seguridad
Si el servidor requiere invitaciones para la sala, el usuario no podrá volver a entrar sin una nueva invitación.

## Banear a un Usuario
**Endpoint:** `[POST] /_matrix/client/v3/rooms/{roomId}/ban`

> **Propósito:** Expulsa al usuario (si estaba dentro) y le prohíbe volver a entrar o ser invitado hasta que se revoque el ban.

### Aplicación en el Proyecto PRADO
Herramienta de moderación para profesores. Si un usuario tiene un comportamiento gravemente inapropiado en el chat general de una asignatura, este comando bloquea su acceso permanentemente al canal, independientemente de su estado de matrícula.

### Parámetros Clave (Payload y Ruta)
* `roomId`: (String) [Ruta] El ID interno de la sala.
* `user_id`: (String) [Payload] El MXID del usuario infractor.
* `reason`: (String) [Payload] Razón del baneo, visible en el registro de moderación.

### Notas de Implementación / Seguridad
Requiere un nivel de poder alto en la sala (normalmente asignado a Administradores o Moderadores).

---

## Levantar el Baneo (Unban) a un Usuario
**Endpoint:** `[POST] /_matrix/client/v3/rooms/{roomId}/unban`

> **Propósito:** Elimina la restricción de baneo de un usuario.

### Aplicación en el Proyecto PRADO
Si la expulsión de un alumno fue temporal (ej. sanción de una semana), el panel o el profesor utilizarán este endpoint para permitir que el alumno pueda ser reinvitado a la clase virtual.

### Parámetros Clave (Payload y Ruta)
* `roomId`: (String) [Ruta] El ID interno de la sala.
* `user_id`: (String) [Payload] El MXID del usuario a desbanear.
* `reason`: (String) [Payload] Razón del levantamiento de la sanción.

### Notas de Implementación / Seguridad
No une automáticamente al usuario a la sala; simplemente permite que la acción de invitar vuelva a funcionar.
