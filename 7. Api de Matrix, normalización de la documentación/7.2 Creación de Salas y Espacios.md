
Esta sección documenta los mecanismos para generar nuevos entornos de comunicación (tanto salas de chat individuales/grupales como Espacios organizacionales) y consultar su árbol jerárquico. En la arquitectura de Matrix, un Espacio es técnicamente una sala convencional configurada con un evento de creación específico, lo que permite unificar el proceso de creación bajo un mismo conjunto de endpoints.

## Recuperar la Jerarquía de un Espacio

**Endpoint:** `[GET] /_matrix/client/v1/rooms/{roomId}/hierarchy`

> **Propósito:** Realiza una paginación en profundidad (depth-first) sobre el árbol de un espacio para localizar todas las salas e infraestructuras secundarias que dependen de él.

### Aplicación en el Proyecto PRADO

El panel de administración utilizará este endpoint para auditar de forma síncrona la estructura de Matrix. Permite consultar al servidor qué salas específicas (de tutorías, subgrupos o laboratorios) están anidadas actualmente dentro del Espacio principal de una asignatura concreta, garantizando que la estructura reflejada en Moodle coincida con la de Matrix.

### Parámetros Clave (Query y Ruta)

- `roomId`: (String) [Ruta] El ID único de la sala de tipo Espacio de la cual se quiere extraer la jerarquía.
    
- `suggested_only`: (Boolean) [Query] Flag opcional (por defecto `false`). Si es `true`, el servidor solo devolverá aquellas salas hijas que hayan sido marcadas explícitamente como sugeridas.
    
- `limit`: (Integer) [Query] Número máximo de salas a incluir por respuesta para evitar sobrecarga del servidor.
    
- `max_depth`: (Integer) [Query] Límite de profundidad jerárquica para la exploración del árbol (evita bucles en espacios con anidamiento complejo).
    
- `from`: (String) [Query] Token de paginación (`next_batch`) obtenido de una respuesta anterior para continuar recuperando elementos.
    

### Notas de Implementación / Seguridad

Requiere autenticación mediante Token de acceso. El endpoint solo procesa eventos de estado del tipo `m.space.child`. Si hay salas hijas en servidores federados externos, el servidor local contactará con los servidores listados en el array `via` para completar la información. Devuelve los datos de las salas a las que el usuario tiene permitido acceder o visualizar.

## Crear una Nueva Sala o Espacio

**Endpoint:** `[POST] /_matrix/client/v3/createRoom`

> **Propósito:** Crea una nueva sala o espacio en el servidor aplicando configuraciones básicas, políticas de visibilidad, alias, niveles de poder e invitaciones iniciales en una única transacción.

### Aplicación en el Proyecto PRADO

Es el motor principal para el despliegue estructural del puente. Cuando el panel detecte una nueva asignatura en Moodle, llamará a este endpoint. Para crear el "Espacio" contenedor de la asignatura, se inyectará el tipo `m.space` dentro de `creation_content`. Posteriormente, se volverá a invocar sin este atributo para crear los canales de chat individuales (teoría, prácticas), vinculándolos al espacio padre.

### Parámetros Clave (Payload)

- `creation_content`: (Object) Claves adicionales para el evento base `m.room.create`. Para crear un Espacio, se debe incluir `type: "m.space"` dentro de este objeto.
    
- `visibility`: (Enum) Define si la sala aparece en el directorio público del servidor (`public` o `private`). En el entorno académico, será `private`.
    
- `room_alias_name`: (String) La parte local del alias deseado (ej: "ia_grupo_a"), que generará un alias canónico `#ia_grupo_a:dominio.es`.
    
- `name`: (String) Nombre descriptivo de la sala (ej: "Inteligencia Artificial - Grupo A").
    
- `topic`: (String) Descripción o tema general de la sala.
    
- `preset`: (Enum) Configuración rápida de reglas de unión y visibilidad. Valores comunes: `private_chat`, `public_chat`, `trusted_private_chat`.
    
- `initial_state`: (Array de Objetos) Lista de eventos de estado a forzar en la creación. Útil para vincular la sala a un Espacio padre desde el momento cero usando un evento `m.space.parent`.
    
- `invite`: (Array de Strings) Lista de Matrix IDs (`@usuario:ugr.es`) a invitar automáticamente en la creación.
    
- `is_direct`: (Boolean) Si es `true`, marca la sala como un mensaje directo 1 a 1.
    
- `power_level_content_override`: (Object) Permite alterar los niveles de poder por defecto de la sala antes de su creación, asignando permisos administrativos a los profesores.
    

### Notas de Implementación / Seguridad

Requiere autenticación mediante Token de acceso. El servidor aplica un orden de resolución estricto: primero genera el evento de creación, luego el de membresía para el creador, aplica los niveles de poder, establece el alias y finalmente procesa las invitaciones. (Nota: Se omite el uso del parámetro `invite_3pid` documentado por la API estándar, ya que la integración PRADO no requerirá servidores de identidad de terceros).
