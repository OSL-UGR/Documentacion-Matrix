

Esta sección documenta cómo Matrix maneja las conversaciones ramificadas (hilos) y las relaciones entre eventos. En canales de asignaturas con decenas o cientos de alumnos, los hilos son esenciales para mantener el orden, permitiendo debatir dudas específicas sin saturar el chat principal. Es un componente fundamental para que los scripts y bots (como un FAQBot) puedan leer, analizar y responder dentro del contexto adecuado de una tutoría.

## Obtener la Lista de Hilos de una Sala

**Endpoint:** `[GET] /_matrix/client/v1/rooms/{roomId}/threads`

> **Propósito:** Recupera una lista paginada de todos los mensajes raíz (thread roots) que han iniciado un hilo de conversación dentro de una sala concreta.

### Aplicación en el Proyecto PRADO

Permite al bot de la asignatura o al panel de control realizar un escaneo rápido para identificar discusiones activas en el chat general sin tener que descargar y procesar todos los mensajes individuales sueltos. Es muy útil para extraer métricas de participación o para que un sistema automatizado busque hilos de dudas que aún no han sido respondidos por el equipo docente.

### Parámetros Clave (Ruta y Query)

- `roomId`: (String) [Ruta] El ID interno de la sala donde se encuentran los hilos.
    
- `include`: (Enum) [Query] Filtro de interés. Valores permitidos: `all` (devuelve todos los hilos de la sala, opción por defecto) o `participated` (devuelve únicamente los hilos en los que el usuario o bot autenticado ha intervenido).
    
- `limit`: (Integer) [Query] Número máximo de raíces de hilo a devolver por petición para evitar sobrecargar la memoria.
    
- `from`: (String) [Query] Token de paginación para solicitar el siguiente lote de resultados.
    

### Notas de Implementación / Seguridad

Requiere autenticación mediante Token de acceso. La respuesta devuelve un array `chunk` con los eventos raíz, los cuales incluyen internamente agregaciones (como el evento más reciente publicado dentro de ese hilo) para facilitar su ordenación cronológica en la interfaz o en el log del servidor.

## Obtener las Respuestas (Eventos Hijos) de un Mensaje

**Endpoint:** `[GET] /_matrix/client/v1/rooms/{roomId}/relations/{eventId}`

> **Propósito:** Recupera todos los eventos secundarios (respuestas) que apuntan y están vinculados directamente a un evento padre específico (el mensaje que inició el hilo).

### Aplicación en el Proyecto PRADO

Es el mecanismo de lectura profunda. Una vez que el sistema detecta un hilo (por ejemplo, iniciado por un alumno preguntando sobre la "Práctica 3"), utiliza este endpoint pasándole el ID de ese primer mensaje para leer toda la conversación anidada debajo. Esto permite que el bot analice el contexto completo antes de emitir una respuesta automática, o que el panel de administración archive esa tutoría específica como un bloque estructurado en la base de datos de Moodle.

### Parámetros Clave (Ruta y Query)

- `roomId`: (String) [Ruta] El ID interno de la sala.
    
- `eventId`: (String) [Ruta] El ID del mensaje original (padre) cuyas respuestas se desean obtener.
    
- `dir`: (Enum) [Query] Dirección de la búsqueda. `f` (hacia adelante, en orden cronológico normal) o `b` (hacia atrás, en orden cronológico inverso, que es el comportamiento por defecto).
    
- `recurse`: (Boolean) [Query] Si es `false` (por defecto), solo incluye respuestas directas al mensaje padre. Si es `true`, el servidor descenderá recursivamente (hasta 3 niveles de profundidad) para recuperar eventos vinculados indirectamente.
    
- `from` / `to`: (String) [Query] Tokens de paginación para acotar el rango de eventos a extraer.
    
- `limit`: (Integer) [Query] Tamaño máximo del bloque de respuestas a devolver.
    

### Notas de Implementación / Seguridad

Requiere autenticación con Token de acceso. La respuesta JSON proporciona un array `chunk` con todos los eventos hijo ordenados topológicamente, además de los tokens `next_batch` y `prev_batch` para controlar la paginación a través del hilo. Si el hilo no tiene respuestas, el array se devuelve vacío.
