
Esta sección abarca la validación de registros, vinculación de identificadores de terceros (3PID como correos o teléfonos) y la administración básica de la cuenta. Es fundamental para el sistema de aprovisionamiento automatizado, permitiendo vincular la identidad del alumno en PRADO con su cuenta en Matrix.

---

## Comprobar Validez de Token de Registro
**Endpoint:** `[GET] /_matrix/client/v1/register/m.login.registration_token/validity`

> **Propósito:** Consulta al servidor si un token de registro proporcionado sigue siendo válido en el momento exacto de la petición.

### Aplicación en el Proyecto PRADO
Si decidimos que el alta de los alumnos no sea 100% silenciosa, sino mediante un enlace de registro generado por el panel de administración, este endpoint permite al panel verificar si el enlace (token) que se le envió al alumno ha caducado antes de mostrarle el formulario.

### Parámetros Clave (Query)
* `token`: (String) El token de registro que se desea validar.

### Notas de Implementación / Seguridad
No requiere autenticación. El servidor devuelve un objeto JSON con el valor booleano `valid`. Los servidores suelen aplicar límites de peticiones (rate limiting) a este endpoint para evitar ataques de fuerza bruta.

## Listar Identificadores de Terceros (3PID)
**Endpoint:** `[GET] /_matrix/client/v3/account/3pid`

> **Propósito:** Obtiene una lista de los identificadores externos (correos electrónicos o números de teléfono) que el servidor local de Matrix tiene asociados a la cuenta del usuario actual.

### Aplicación en el Proyecto PRADO
Útil para que el panel de administración compruebe si la cuenta de un estudiante tiene correctamente vinculado su correo institucional (`@correo.ugr.es`). Esta vinculación es necesaria para que el alumno pueda recuperar su contraseña de forma autónoma.

### Parámetros Clave
* No requiere parámetros en el cuerpo de la petición. 

### Notas de Implementación / Seguridad
Requiere autenticación (Token de acceso). Devuelve un array de objetos con el medio (`email` o `msisdn`), la dirección y las fechas de validación.

---

## Añadir Identificador a la Cuenta (3PID)
**Endpoint:** `[POST] /_matrix/client/v3/account/3pid/add`

> **Propósito:** Añade información de contacto confirmada (email o teléfono) a la cuenta del usuario para que el servidor pueda usarla en reseteos de contraseña.

### Aplicación en el Proyecto PRADO
Se utilizará en el script de sincronización para inyectar y vincular directamente el correo oficial de la UGR extraído de Moodle al perfil de Matrix del usuario.

### Parámetros Clave (Payload)
* `auth`: (Object) Información adicional para la API de Autenticación Interactiva del Usuario (UIA). Incluye la clave de sesión (`session`) y el tipo de autenticación (`type`).
* `client_secret`: (String) El secreto de cliente utilizado en la sesión.
* `sid`: (String) El identificador de sesión proporcionado previamente por el servidor durante el proceso de solicitud de token.

### Notas de Implementación / Seguridad
Requiere autenticación y soporta Autenticación Interactiva del Usuario (UIA). La documentación advierte que no puede utilizarse si el token de acceso se obtuvo mediante la API OAuth 2.0. El cliente debe verificar la capacidad `m.3pid_changes` antes de usarlo.

## Solicitar Validación de Email
**Endpoint:** `[POST] /_matrix/client/v3/account/3pid/email/requestToken`

> **Propósito:** Inicia el proceso para vincular un email a la cuenta ordenando al servidor que envíe un mensaje con un token de validación a dicha dirección.

### Aplicación en el Proyecto PRADO
Es el paso previo obligatorio si un usuario decide añadir una dirección de correo alternativa desde su cliente (Element). El servidor enviará un correo para comprobar que el alumno tiene acceso a la bandeja de entrada.

### Parámetros Clave (Payload)
* `email`: (String) La dirección de correo a validar.
* `client_secret`: (String) Cadena única generada por el cliente para identificar el intento de validación.
* `send_attempt`: (Integer) Contador para evitar el envío repetido de correos. El servidor solo enviará un nuevo correo si este número es superior al anterior registrado.

### Notas de Implementación / Seguridad
No requiere autenticación. Devuelve un identificador de sesión (`sid`) que debe guardarse para completar el proceso en el endpoint `/3pid/add`.

## Desactivar Cuenta de Usuario
**Endpoint:** `[POST] /_matrix/client/v3/account/deactivate`

> **Propósito:** Desactiva permanentemente la cuenta de un usuario, impidiendo futuros inicios de sesión y opcionalmente borrando todo rastro de sus mensajes.

### Aplicación en el Proyecto PRADO
Fundamental para el ciclo de vida de los usuarios. Cuando un estudiante pierde su matrícula, se gradúa o su cuenta expira en PRADO, el script ejecutará este endpoint para revocar su acceso al chat de la universidad. Se ejecutará *sin* la opción de borrado (`erase: false`) para preservar el historial de mensajes útil (dudas resueltas, etc.) en las salas de las asignaturas.

### Parámetros Clave (Payload)
* `auth`: (Object) Información de la API de Autenticación Interactiva (UIA).
* `erase`: (Boolean) Si es `true`, borra el contenido enviado por el usuario (redacción). Para PRADO, debe ser `false` por defecto.

### Notas de Implementación / Seguridad
Requiere autenticación y el uso de la API de Autenticación Interactiva (UIA). Devuelve el resultado de la desvinculación de los 3PIDs asociados en servidores de identidad.

## Cambiar Contraseña
**Endpoint:** `[POST] /_matrix/client/v3/account/password`

> **Propósito:** Modifica la contraseña de la cuenta del usuario autenticado en el servidor local.

### Aplicación en el Proyecto PRADO
Permite que un usuario cambie su contraseña desde la interfaz de Element si su cuenta se creó de forma aislada. Sin embargo, si se integra PRADO y Matrix mediante SSO (Single Sign-On, como SAML/CAS), la contraseña se gestionará en Moodle y este endpoint podría ser bloqueado o quedar sin uso.

### Parámetros Clave (Payload)
* `new_password`: (String) La nueva contraseña para la cuenta.
* `logout_devices`: (Boolean) Si es `true` (por defecto), cierra la sesión en todos los demás dispositivos tras el cambio.
* `auth`: (Object) Información de la API de Autenticación Interactiva (UIA).

### Notas de Implementación / Seguridad
Requiere autenticación y el uso de la API de Autenticación Interactiva (UIA). El servidor no debe revocar el token de acceso de la sesión desde la que se realiza la petición, pero sí los demás si `logout_devices` es `true`.

## Solicitar Token para Restablecer Contraseña (Email)
**Endpoint:** `[POST] /_matrix/client/v3/account/password/email/requestToken`

> **Propósito:** Inicia el proceso de recuperación de contraseña comprobando si el email existe en el sistema y enviando un enlace o código de recuperación.

### Aplicación en el Proyecto PRADO
Es el mecanismo de recuperación de acceso "Olvidé mi contraseña" para los alumnos, siempre y cuando no se esté utilizando un sistema SSO centralizado. El panel no interactúa directamente con él, lo usa el cliente web (Element).

### Parámetros Clave (Payload)
* `email`: (String) El correo institucional donde enviar la recuperación.
* `client_secret`: (String) Identificador único del intento de recuperación.
* `send_attempt`: (Integer) Contador para gestionar reintentos.

### Notas de Implementación / Seguridad
No requiere autenticación previa. Devuelve un `sid` (ID de sesión) para completar el proceso de cambio de clave.

## Registrar Nueva Cuenta de Usuario
**Endpoint:** `[POST] /_matrix/client/v3/register`

> **Propósito:** Registra un nuevo usuario en el servidor (homeserver), generando un Matrix ID (MXID) y devolviendo un token de acceso inicial.

### Aplicación en el Proyecto PRADO
**El endpoint más importante de la sección.** El panel de administración usará este método, ejecutado a través de credenciales de administrador (o delegación), para crear las cuentas de los alumnos (`@matricula:ugr.es`) de forma automatizada y sin intervención del estudiante.

### Parámetros Clave (Payload)
* `username`: (String) La parte local deseada para el Matrix ID (ej: "juanperez").
* `password`: (String) Contraseña inicial de la cuenta.
* `inhibit_login`: (Boolean) Si es `true`, crea la cuenta pero no devuelve un token de sesión (útil para creación en lote).
* `auth`: (Object) Información de la API de Autenticación Interactiva (UIA).
* `kind`: (Query String Enum) Tipo de cuenta, por defecto `user`.

### Notas de Implementación / Seguridad
El servidor puede forzar el uso de la API UIA. Devuelve el `user_id` completo y el `access_token` (si `inhibit_login` es `false`). Si se usa para la integración de "Application Services" (como puentes oficiales), puede requerir permisos especiales.

## Comprobar Disponibilidad de Nombre de Usuario
**Endpoint:** `[GET] /_matrix/client/v3/register/available`

> **Propósito:** Verifica si un nombre de usuario (parte local del MXID) propuesto está disponible para su registro.

### Aplicación en el Proyecto PRADO
Paso previo de validación. Antes de que el script ejecute el registro masivo, consultará este endpoint para evitar conflictos si dos alumnos tienen identificadores institucionales idénticos o si la cuenta ya fue creada en cuatrimestres anteriores.

### Parámetros Clave (Query)
* `username`: (String) El nombre de usuario que se desea registrar.

### Notas de Implementación / Seguridad
No requiere autenticación. Devuelve un valor booleano (`available`). La respuesta es indicativa; no reserva el nombre de usuario de forma permanente.

## Solicitar Validación de Email para Registro
**Endpoint:** `[POST] /_matrix/client/v3/register/email/requestToken`

> **Propósito:** Inicia la validación de un correo electrónico durante el proceso de registro manual de una nueva cuenta.

### Aplicación en el Proyecto PRADO
Solo aplicable si se decide permitir el registro abierto manual (donde el alumno se registra él mismo usando su correo de la UGR). Si las cuentas las crea el script de sincronización desde PRADO, este endpoint se omite, ya que la identidad ya ha sido verificada por Moodle.

### Parámetros Clave (Payload)
* `email`: (String) Correo a validar.
* `client_secret`: (String) Identificador único del cliente.
* `send_attempt`: (Integer) Contador de reintentos.

### Notas de Implementación / Seguridad
No requiere autenticación. Devuelve el `sid` necesario para confirmar la identidad antes de ejecutar la llamada final a `/register`.
