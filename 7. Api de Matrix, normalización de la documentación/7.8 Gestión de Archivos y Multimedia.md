

Esta sección documenta la interacción con el repositorio de contenido de Matrix (Content Repository). En la arquitectura de Matrix, los archivos (imágenes, PDFs, avatares) no se envían directamente dentro de los mensajes JSON. Primero deben subirse a este repositorio, el cual devuelve un identificador único (`mxc://`), y posteriormente ese identificador es el que se incluye en el mensaje o en el perfil del usuario.

## Consultar la Configuración del Repositorio

**Endpoint:** `[GET] /_matrix/client/v1/media/config`

> **Propósito:** Recupera la configuración pública del repositorio de archivos del servidor, específicamente las limitaciones de subida.

### Aplicación en el Proyecto PRADO

Paso de validación previa. Si un profesor adjunta una guía docente o un dataset muy pesado en Moodle y el script de sincronización intenta replicarlo en Matrix, el panel consultará primero este endpoint. Esto permite comprobar el límite de megabytes (`m.upload.size`) y alertar al profesor o comprimir el archivo antes de que la subida falle por exceso de tamaño.

### Parámetros Clave

- No requiere parámetros en la ruta ni en el cuerpo.
    

### Notas de Implementación / Seguridad

Requiere autenticación mediante Token de acceso. Devuelve un objeto JSON con la propiedad `m.upload.size` (el tamaño máximo en bytes). Si el valor es nulo o no aparece, se asume que el límite es desconocido o gestionado por un proxy inverso (como Nginx).

## Subir Archivo al Repositorio

**Endpoint:** `[POST] /_matrix/media/v3/upload`

> **Propósito:** Sube un archivo binario al servidor y genera un identificador de recursos URI propio de Matrix (`mxc://`).

### Aplicación en el Proyecto PRADO

El motor de transferencia de ficheros. Se utilizará para múltiples propósitos:

1. Subir las fotos de perfil de los estudiantes extraídas de Moodle antes de llamar al endpoint de actualización de perfil.
    
2. Subir PDFs, imágenes o documentos adjuntos a los foros de PRADO para que el bot pueda enviarlos como mensajes a las salas de las asignaturas.
    

### Parámetros Clave (Query y Headers)

- `filename`: (String) [Query] El nombre del archivo que se está subiendo (ej: "Guia_Docente.pdf").
    
- `Content-Type`: (String) [Header] El tipo MIME del archivo (ej: `application/pdf` o `image/jpeg`). Si no se especifica, el servidor asume `application/octet-stream`.
    
- El cuerpo de la petición (Payload) no es un JSON, sino los bytes puros del archivo (`application/octet-stream`).
    

### Notas de Implementación / Seguridad

Requiere autenticación con Token de acceso. Devuelve un objeto JSON con la clave `content_uri` (ej: `mxc://ugr.es/ascERGshawAWawugaAcauga`). Este URI es el que se debe guardar en la base de datos o enviar en los mensajes.

## Descargar Archivo del Repositorio

**Endpoint:** `[GET] /_matrix/client/v1/media/download/{serverName}/{mediaId}`

> **Propósito:** Convierte un identificador interno de Matrix (`mxc://`) en el archivo binario real para su visualización o descarga.

### Aplicación en el Proyecto PRADO

Permite el flujo inverso. Si un estudiante envía una imagen o un archivo de texto al chat de tutorías en Matrix, y el profesor quiere archivarlo en el panel de Moodle, el script capturará el URI del mensaje, extraerá el nombre del servidor y el ID, e invocará este endpoint para descargar los bytes del archivo al servidor de la universidad.

### Parámetros Clave (Ruta y Query)

- `serverName`: (String) [Ruta] El dominio del servidor alojador, extraído de la primera parte del URI (ej: `ugr.es`).
    
- `mediaId`: (String) [Ruta] El identificador alfanumérico del archivo, extraído de la segunda parte del URI.
    
- `timeout_ms`: (Integer) [Query] Tiempo máximo en milisegundos que el cliente está dispuesto a esperar si el archivo aún se está propagando por la red federada (por defecto 20000 ms).
    

### Notas de Implementación / Seguridad

Requiere autenticación mediante Token de acceso (las versiones más recientes de Matrix exigen autenticación para descargar archivos y evitar abusos del servidor). La respuesta no es un JSON, sino el archivo binario (`application/octet-stream`), y las cabeceras HTTP (como `Content-Disposition`) indicarán el nombre original del archivo y si debe mostrarse en línea o descargarse como adjunto.
